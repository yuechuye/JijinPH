# 周涨幅排名优化 + 动量评分系统设计

## 概述

对现有基金周涨幅排行榜系统进行两个优化：
1. **计算窗口调整**：从「周一→周五」改为「周五→周四」，使排名生成与买入执行之间的间隔从2-3天压缩到1晚
2. **新增动量评分**：用多周期收益率加权（1周/2周/4周/12周），为每只ETF算出动量得分，辅助跨主题对比

## 策略背景

- **策略类型**：动量/追涨（趋势延续）
- **操作模式**：每周轮动（排名变了就调仓）
- **买入时机**：周四收盘后跑脚本，周五开盘/盘中买入
- **数据源**：api.fund.eastmoney.com 净值接口（akshare `fund_etf_fund_info_em`）

---

## 阶段一：计算窗口调整为「周五→周四」

### 理由

- 排名和买入之间间隔越短，排名信号被后续事件（尤其是周末消息面）"污染"的概率越低
- 周五往往是机构"做净值"的日子，动量板块在周五容易继续被推高
- 周四晚上跑脚本比周末跑更紧凑，操作节奏更清晰

### 改动点

**`get_trading_dates()` → 改为 `get_momentum_dates()`**

返回多时间点 dict，其中阶段一使用 `friday_before` 和 `thursday`：

```
friday_before = 最近的上一个周五
thursday      = 最近的上一个周四
```

**`get_week_range()` 返回 `f"{last_friday} ~ {last_thursday}"`**

**`fetch_one_etf_weekly()` → 扩展为 `fetch_one_etf_momentum()`**

暂不详述（见阶段二）。

### 影响范围

- `cli.py`：两个函数修改，约10行
- 不影响 `script.js` / `index.html`（它们只读 `week` 字符串）

## 阶段二：动量评分系统

### 理由

- 单周涨幅容易受偶然事件干扰，多周期加权后只有「短期在涨 + 中期也在涨」的ETF才能拿高分
- 学术验证充分：截面动量（cross-sectional momentum）的经典框架就是多周期收益加权
- 输出两套数据：主题内排名（按周涨幅） + 跨主题动量总榜（按动量得分），互不冲突

### 加权公式

```
动量得分 = 1周回报×40% + 2周回报×30% + 4周回报×20% + 12周回报×10%
```

权重向近期倾斜，体现动量策略核心——越近的趋势越重要，但远期趋势提供方向确认。

### 净值时间点

共取5个时间点（均在周四，确保与阶段一窗口对齐）：

| 标记  | 含义       | 示例日期    |
|--------|------------|-------------|
| T0     | 本周四     | 2026-07-16  |
| T-1    | 1周前（上周四） | 2026-07-09  |
| T-2    | 2周前      | 2026-07-02  |
| T-4    | 4周前      | 2026-06-18  |
| T-12   | 12周前     | 2026-04-23  |

### 周期回报计算

```
1周回报  = (T0净值 / T-1净值 - 1) × 100%
2周回报  = (T0净值 / T-2净值 - 1) × 100%
4周回报  = (T0净值 / T-4净值 - 1) × 100%
12周回报 = (T0净值 / T-12净值 - 1) × 100%
```

### 缺失数据处理

- 如果ETF成立不足12周，缺失周期的权重按比例重新分配给可用周期
- 如果某时间点净值缺失（如非交易日），取最近一个交易日的净值
- 如果可用周期 < 2，该ETF不参与动量评分（返回 None）

### 影响范围

- `cli.py`：`fetch_one_etf_momentum()` 新增约40行，`build_result()` 扩展约10行
- `script.js` / `index.html`：新增"动量总榜"Tab，约30行

## 输出格式

### JSON（`latest.json` / `weekly/*.json`）

```json
{
  "week": "2026-07-10 ~ 2026-07-16",
  "updatedAt": "2026-07-16T20:30:00",
  "themes": [
    {
      "name": "半导体/芯片",
      "funds": [
        {
          "code": "512480",
          "name": "半导体ETF",
          "weeklyReturn": 5.23,
          "momentumScore": 3.87,
          "returns": {
            "1w": 5.23,
            "2w": 3.45,
            "4w": 2.10,
            "12w": -1.20
          }
        }
      ]
    }
  ],
  "momentumRanking": [
    { "code": "512480", "name": "半导体ETF", "theme": "半导体/芯片", "momentumScore": 3.87 },
    { "code": "159995", "name": "芯片ETF", "theme": "半导体/芯片", "momentumScore": 3.52 }
  ]
}
```

- `themes[].funds` 按 `weeklyReturn` 降序（主题内排名）
- `momentumRanking` 按 `momentumScore` 降序（跨主题总榜）
- `returns` 拆开保存，前端可展示各周期明细

### 终端

保持现有简洁摘要格式，新增一段动量总榜Top5的简要输出。

### 前端

在 `index.html` 新增一个"动量榜"Tab，展示跨主题动量前N名及各周期回报明细。
